Introduction

MultiBoost package is a fast C++ implementation of multi-class/multi-label/multi-task boosting algorithms. It is based on AdaBoost.MH but also implements popular cascade classifiers and FilterBoost. The package contains common multi-class base learners (stumps, trees, products, Haar filters). Further base learners and strong learners following the boosting paradigm can be easily implemented in a flexible framework. Feel free to use it and to modify it!

This software is covered by the LGPL licence.

Installation

Execute the following command in the directory where the zipped source code is located in order to compile :

% unzip multiboost.zip
% cmake .
% make

Note: you need cmake to create the Makefile. To run the program, just type:

% multiboost

To get some help, type:

% multiboost -help

Coding Informations

MultiBoost was built with Boosting algorithms in mind, in particular for multi-class AdaBoost.MH.

The concept has been centered on the base learner (aka weak learner), which holds the methods to find the best function h(x) at each boosting iteration. Each base function can also decide the type of data to process, and even the strong learner that is calling it.

Here is how it works:

  • Each base learner (i.e. SingleStumpLearner) is registered in a static list, that is a class factory, using just a simple macro (see REGISTER_LEARNER for details).
  • The users selects the base learner either by providing an argument on the command line or by loading a strong hypothesis file.
  • If the base learner is in the static list, it will inform the framework what are its InputData and GenericStrongLearner. If the default data format (InputData) and strong learner (AdaBoostMHLearner) are not enough they can be overrided with two methods of BaseLearner: createInputData() and createGenericStrongLearner().
  • The strong learner run the boosting process by creating an object of type BaseLearner (any registered weak learner must implement the method subCreate() which returns an allocated copy of itself), that will have to find the best h(x) for the current boosting iteration. Once the function has been received it is stored in the vector of the weak learner. Note that the weighting factor alpha (if needed) needs to be stored in the weak learner.
  • The serialization is performed simply by overriding the proper classes of BaseLearner.
  • The classification depends on the strong learner chosen. can classify any type of weak learner that use AdaBoost.MH as strong learner, as long as they implement all the abstract methods.

For any other question, please refer to the documentation or contact one of the authors.
